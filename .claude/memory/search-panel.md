# Search sub-panel (flow-native canvas search)

Canvas text search lives in flow's OWN controls dock as a **Search sub-panel**
(Style·Stroke·Text·Align·**Search**·Layers), NOT Excalidraw's native sidebar
(which is fully retired). Running search from the [[bottom-bar]] auto-opens the
panel. Spec: `docs/superpowers/specs/2026-07-09-search-panel-design.md`. Shipped
2026-07-09. Replaces the earlier native-sidebar bridge approach.

## Architecture
- **Fork export #2** `getSearchMatches(query, elements)` (after `executeAction`):
  `vendor/.../packages/excalidraw/searchMatches.ts` (NEW, additive — helpers
  duplicated from SearchMenu.tsx rather than refactoring it, to keep the diff
  additive/low-conflict) + one export line in `index.tsx`. Returns
  `{ id, index, preview, matchedLines }[]` reusing Excalidraw's own
  `measureText`/`getFontString` so highlight geometry is exact.
- flow sets `appState.searchMatches` via the public `updateScene` → the
  interactive canvas renders highlights with NO Excalidraw UI. **Verified**: the
  focused match renders orange, others yellow, exactly like native search.
- Jump-to-match = `api.scrollToContent(textEl, { fitToContent, animate })`.

## Shipped (flow)
- `src/lib/search-matches.ts`: typed wrapper (`findMatches`/`applyMatches`/
  `clearMatches`/`scrollToMatch`) + re-exports `SearchResult`.
- `src/excalidraw-fork.d.ts`: module AUGMENTATION for the new export (dist .d.ts
  isn't regenerated by the esbuild build). **Gotcha**: the file MUST contain
  `export {}` or the `declare module` SHADOWS the whole package (breaks every
  other import) instead of merging.
- `src/ui/panels/SearchPanel.tsx`: input + count + prev/next + clickable results
  list (preview with matched substring emphasised); owns focusIndex. Driven by a
  `SearchSignal { query: string | null, nonce }` — string = adopt+run
  (bottom-bar), null = open+focus only (Ctrl/Cmd+F).
- `panel-dock-state.ts`: new `openSub` action (visible+expanded, un-collapse dock;
  leaves a torn-off panel floating). `PanelDock` takes `openPanel={id,nonce}` and
  opens on nonce bump.
- `PanelsRoot`: registers the `search` panel + passes `openPanel`.
- `App.tsx`: owns `search` signal (not persisted); bottom bar `onSearch` bumps it;
  capture-phase Ctrl/Cmd+F listener repoints the shortcut to flow's panel
  (preventDefault+stopPropagation before Excalidraw's handler).
- Bottom bar: `SearchControl.onExecute` → App `onSearch` (no longer the bridge).
- `index.css`: `.default-sidebar` + trigger fully hidden again (native search
  retired). **Deleted** `src/lib/search-bridge.ts`(+test).

## Build / fork
- Rebuild vendor dist after fork edits: from `vendor/excalidraw/packages/excalidraw`
  run `node ../../scripts/buildPackage.js` under **Node 22** (nvm; yarn/npm build
  blocked on Node 25). esbuild only — does NOT emit `.d.ts` (hence the flow-side
  augmentation). Fork source committed to submodule `bobtherobot/excalidraw`
  branch `flow`; pointer bumped in flow.
- flow's own `dist/` (committed) bundles the vendor via Vite, so the deployed app
  carries getSearchMatches even though the vendor dist is gitignored.
